---
description: Диагностика пайплайна сбора рейсов (russia-flights) — GitHub Actions + резервный cron на VPS
---

Проверь состояние пайплайна обновления данных по шагам:

1. Через `gh run list --limit 5` посмотри последние запуски workflow.
2. Если последний запуск зафейлился — вытащи лог через `gh run view <id> --log` и определи,
   на каком источнике (AirLabs / AeroDataBox / OpenSky / Яндекс.Расписания) или на каком
   стейдже (Stage1/Stage2) произошёл сбой.
3. Подключись к `timeweb-flights-vps` по SSH и проверь, отработал ли резервный cron в 03:00 МСК
   (curl-пул), и не разошлись ли версии `routes.json` на хостинге и в репозитории.
4. Сравни количество маршрутов в текущем `routes.json` с предыдущим коммитом — если просадка
   аномально большая, не предлагай автоматический коммит, сначала покажи диагностику и причину.
5. Предложи план исправления в виде текста. НЕ делай `git push` и НЕ перезапускай workflow
   без явного подтверждения — эти действия в settings.json стоят в списке "ask".
