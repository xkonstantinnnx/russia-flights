# ✈ Прямые рейсы из России · интерактивная карта

Карта автоматически обновляется каждую неделю, получая актуальные данные
о рейсах из [OpenSky Network](https://opensky-network.org/).

## Структура файлов

```
├── index.html          # Карта (открывается в браузере)
├── routes.json         # Данные о маршрутах (обновляются автоматически)
├── update_routes.py    # Скрипт обновления
└── .github/
    └── workflows/
        └── update.yml  # Расписание автообновления
```

---

## Первоначальная настройка

### 1. Создайте форк репозитория на GitHub

Нажмите **Fork** в правом верхнем углу страницы репозитория.

### 2. Зарегистрируйтесь на OpenSky Network

Перейдите на [opensky-network.org](https://opensky-network.org/) и создайте
бесплатный аккаунт. Это необходимо для увеличения лимита запросов API.

### 3. Добавьте секреты в GitHub

В вашем форке откройте:  
**Settings → Secrets and variables → Actions → New repository secret**

Добавьте два секрета:

| Имя            | Значение                    |
|----------------|-----------------------------|
| `OPENSKY_USER` | Ваш логин на opensky-network |
| `OPENSKY_PASS` | Ваш пароль на opensky-network |

### 4. Включите GitHub Pages

**Settings → Pages → Source: Deploy from a branch → Branch: main → / (root)**

Через минуту карта будет доступна по адресу:  
`https://<ваш-логин>.github.io/<имя-репозитория>/`

### 5. Запустите первое обновление вручную

**Actions → Обновление маршрутов → Run workflow**

Скрипт запросит данные за последние 28 дней и сохранит `routes.json`.
Следующие обновления будут происходить автоматически каждый понедельник.

## Как это работает

```
GitHub Actions (каждый понедельник, 04:00 UTC)
    │
    ▼
update_routes.py
    │  Для каждого из ~30 российских аэропортов:
    │  запрашивает вылеты за последние 28 дней (4 окна × 7 дней)
    │  через OpenSky REST API
    │
    ▼
Фильтрация:
    - только международные направления (не Россия)
    - минимум 2 рейса за 28 дней (отсев разовых вылетов)
    - только известные аэропорты назначения
    │
    ▼
routes.json  ←  коммит в репозиторий
    │
    ▼
index.html загружает routes.json при открытии
(при недоступности файла — встроенные резервные данные)
```

## Параметры настройки

В файле `update_routes.py` можно изменить:

- **`MIN_FLIGHTS = 2`** — минимальное количество рейсов за 28 дней для включения маршрута.
  Увеличьте до 4, если хотите видеть только регулярные рейсы (не чартеры).

- **`RU_AIRPORTS`** — добавьте или удалите аэропорты РФ по ICAO-коду.

- **`DEST_INFO`** — добавьте новые аэропорты назначения, если скрипт
  не распознаёт какой-то ICAO-код.

## Лимиты OpenSky API

| Тип доступа | Лимит запросов | Достаточно? |
|-------------|---------------|-------------|
| Анонимный   | ~400/день     | На грани (31 аэропорт × 4 окна = 124 запроса) |
| Авторизованный (бесплатно) | без жёстких ограничений | ✓ |

Именно поэтому рекомендуется использовать авторизованный доступ (секреты).
